Предполагаю разработку CRM системы для подбора персонала.

С учетом того, что предполагается совместить в системе различные сервисы - работа с кандидатами, с заказчиками, отслеживание эффективности персонала и в дальнейшем что то еще,
которые не будут сильно влиять друг на друга, а лишь запрашивать часть данных у смежных сервисов с дальнейшей агрегацией и передачей их пользователю, предполагаю использовать сервисно-ориентированную архитектуру.

Данный вид архитектуры позволит независимо длуг от друга подключать/отключать необходимые блоки в процессе разработки без серьезного влияния на другие части приложения и
более качественно реагировать на потребности заказчика в новых сервисах.
Так же появляется возможность использовать разные технологии для разработки сервисов - Java для серверной части и JS для web,
что с учетом предполагаемой мультиплатформенности системы (десктоп, веб, мобилки) предполагается приемуществом.
